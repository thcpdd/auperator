# 系统架构

Auperator 采用基于消息队列的解耦架构，实现日志采集、传输、分析的完整闭环。

## 整体架构图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Auperator 架构                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────────────────┐  │
│  │  日志采集器   │───▶│  Redis       │───▶│  Agent (消费者)          │  │
│  │  Collector   │    │  Streams     │    │  Consumer                │  │
│  └──────────────┘    └──────────────┘    └──────────────────────────┘  │
│        │                    │                      │                    │
│        ▼                    ▼                      ▼                    │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────────────────┐  │
│  │  日志源       │    │  消息队列    │    │  日志分析                │  │
│  │  - Docker    │    │  - 持久化    │    │  - Bug 检测               │  │
```

## 核心组件

### 1. 日志采集器 (Collector)

负责从各种日志源收集日志，并发送到消息队列。

**组件：**
- **日志源 (Source)**: 从特定来源读取原始日志
- **适配器 (Adapter)**: 将原始日志解析为标准格式
- **发送器 (Sender)**: 将日志发送到 Redis Stream

**特点：**
- 支持多种日志源（Docker、File、Kubernetes）
- 适配器模式解析不同日志格式
- 流式处理，实时采集

### 2. 消息队列 (Redis Streams)

作为采集器和 Agent 之间的缓冲区。

**特点：**
- **持久化**: Redis AOF/RDB 保证数据不丢失
- **解耦**: 采集器和 Agent 独立运行
- **多消费者**: 支持多个 Agent 实例
- **回溯消费**: 可重新消费历史日志

**Stream 结构：**

```
Stream: logs:main

Entry:
  - id: 1709251200000-0
  - data: {"timestamp": "...", "level": "ERROR", "message": "..."}

消费者组：auperator-group
消费者：agent-1, agent-2, ...
```

### 3. Agent (消费者)

从 Redis 消费日志，进行智能分析和修复。

**功能：**
- 日志分析
- Bug 检测
- 根因分析
- 自动修复
- PR 提交

## 数据流

```
1. 日志产生
   │
   ▼
2. 日志源读取 (Source.read())
   │
   ▼
3. 适配器解析 (Adapter.parse())
   │
   ▼
4. 发送到 Redis (Sender.send())
   │
   ▼
5. Agent 消费 (Consumer.consume())
   │
   ▼
6. 分析处理
   │
   ▼
7. 确认消息 (XACK)
```

## 日志源和适配器

### 支持的日志源

| 类型 | 状态 | 描述 |
|------|------|------|
| Docker | ✅ 已实现 | 从 Docker 容器读取日志 |

## 扩展性

### 水平扩展

- 多个采集器实例可以同时向同一个 Stream 发送日志
- 多个 Agent 实例可以通过消费者组模式负载均衡

### 垂直扩展

- 自定义日志源：继承 `BaseLogSource`
- 自定义适配器：继承 `BaseLogAdapter`
- 自定义处理器：实现 `LogHandler`
